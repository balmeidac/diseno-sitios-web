Introducción a la Web

La World Wide Web (WWW) es un sistema de información distribuido que funciona sobre Internet y permite acceder a documentos y recursos a través de navegadores web. Es la forma más popular de acceder a información en la era digital.

Conceptos Fundamentales:

  • Internet: Infraestructura global de redes interconectadas
  • Web: Sistema de información que funciona sobre Internet
  • Sitio Web: Conjunto de páginas web relacionadas entre sí
  • Página Web: Documento digital accesible mediante un navegador
  • URL: Dirección única que identifica cada recurso en la Web
Importante: Aunque comúnmente se usan como sinónimos, Internet y Web son conceptos diferentes. Internet es la infraestructura, mientras que la Web es uno de los servicios que funcionan sobre ella.

Historia y Evolución de la Web

Línea de Tiempo:

1960s - Los Inicios:

  • ARPANET: Precursor de Internet creado por el Departamento de Defensa de EE.UU.
  • Concepto de "red de redes" para comunicación militar y académica

1989 - El Nacimiento de la Web:

  • Tim Berners-Lee propone la World Wide Web en el CERN
  • Crea los tres pilares fundamentales: HTML, HTTP y URL
  • Primera página web publicada: info.cern.ch

1990s - La Expansión:

  • 1991: Primera página web pública
  • 1993: Mosaic, el primer navegador gráfico popular
  • 1995: Netscape Navigator y el inicio de la web comercial
  • 1998: Google revoluciona la búsqueda de información

2000s - Web 2.0 y Redes Sociales:

  • 2004: Facebook y el auge de las redes sociales
  • 2005: YouTube y la revolución del video online
  • 2006: Twitter y la comunicación en tiempo real

2010s - Web Móvil y Aplicaciones:

  • Explosión de smartphones y tablets
  • Diseño responsive como estándar
  • Aplicaciones web progresivas (PWA)

2020s - Web Moderna:

  • Inteligencia Artificial integrada
  • WebAssembly para aplicaciones de alto rendimiento
  • Realidad virtual y aumentada en la web

La World Wide Web (WWW)

La World Wide Web es un sistema de información hipertextual que permite acceder a documentos multimedia a través de Internet. Funciona mediante tres tecnologías fundamentales:

Tecnologías Fundamentales:

1. HTML (HyperText Markup Language)

Lenguaje de marcado para estructurar el contenido de las páginas web.

  • Define la estructura y semántica del contenido
  • Usa etiquetas para organizar la información
  • Es el esqueleto de toda página web

2. HTTP/HTTPS (HyperText Transfer Protocol)

Protocolo de comunicación para transferir datos entre cliente y servidor.

  • Define cómo se realizan las solicitudes y respuestas
  • HTTPS añade una capa de seguridad mediante cifrado
  • Es el lenguaje que hablan navegadores y servidores

3. URL (Uniform Resource Locator)

Dirección única que identifica cada recurso en la Web.

  • Especifica la ubicación exacta de cada recurso
  • Incluye protocolo, dominio y ruta del recurso
  • Permite navegar entre diferentes páginas y sitios

Características Principales:

  • Hipertexto: Documentos conectados mediante enlaces
  • Accesibilidad: Disponible globalmente las 24 horas
  • Interactividad: Permite la interacción usuario-sistema
  • Multiplataforma: Funciona en diferentes dispositivos y sistemas

Cliente y Servidor

El modelo cliente-servidor es la arquitectura fundamental sobre la que funciona la Web. Define cómo se comunican y distribuyen las responsabilidades entre los diferentes componentes.

Componentes del Modelo:

Cliente:

  • Definición: Aplicación que solicita servicios o recursos
  • Ejemplos: Navegadores web (Chrome, Firefox, Safari)
  • Funciones:
    • Enviar solicitudes al servidor
    • Procesar y mostrar las respuestas
    • Manejar la interacción del usuario
    • Ejecutar código del lado del cliente (JavaScript)

Servidor:

  • Definición: Computadora o programa que proporciona servicios
  • Ejemplos: Servidores web (Apache, Nginx)
  • Funciones:
    • Recibir y procesar solicitudes
    • Almacenar y gestionar recursos
    • Enviar respuestas apropiadas
    • Ejecutar código del lado del servidor (PHP, Python, Node.js)

Flujo de Comunicación:

  1. El cliente inicia una solicitud al servidor
  2. El servidor recibe y procesa la solicitud
  3. El servidor envía una respuesta al cliente
  4. El cliente procesa y muestra la respuesta
Ventajas del Modelo Cliente-Servidor: Centralización de datos, mantenimiento simplificado, escalabilidad, seguridad mejorada y distribución eficiente de recursos.

Protocolo HTTP/HTTPS

HTTP (HyperText Transfer Protocol) es el protocolo fundamental que permite la comunicación entre clientes y servidores en la World Wide Web. HTTPS es la versión segura de HTTP.

HTTP (HyperText Transfer Protocol):

  • Puerto: 80 (por defecto)
  • Cifrado: No (datos en texto plano)
  • Velocidad: Ligeramente más rápido
  • Uso: Sitios no sensibles, desarrollo local

HTTPS (HTTP Secure):

  • Puerto: 443 (por defecto)
  • Cifrado: Sí (mediante SSL/TLS)
  • Velocidad: Ligeramente más lento por el cifrado
  • Uso: Sitios con datos sensibles, comercio electrónico

Métodos HTTP Principales:

  • GET: Solicitar recursos (páginas, imágenes, etc.)
  • POST: Enviar datos para crear nuevos recursos
  • PUT: Actualizar recursos existentes
  • DELETE: Eliminar recursos
  • HEAD: Similar a GET pero sin el cuerpo de la respuesta

Códigos de Estado HTTP:

  • 2xx: Respuestas exitosas (200 OK, 201 Created)
  • 3xx: Redirecciones (301 Moved Permanently, 302 Found)
  • 4xx: Errores del cliente (404 Not Found, 403 Forbidden)
  • 5xx: Errores del servidor (500 Internal Server Error)
¿Por qué usar HTTPS? Protección de datos, confianza del usuario, mejor posicionamiento en buscadores, cumplimiento de normativas y acceso a APIs modernas.

Ejercicios Resueltos

Ejercicio 1.1: Análisis del modelo cliente-servidor

Enunciado:

Describe el funcionamiento del modelo cliente-servidor en una petición web. Considera el siguiente escenario: un usuario abre su navegador e ingresa la dirección https://www.ejemplo.com/index.html.

Resolución paso a paso:

Paso 1: Cliente: El navegador (cliente) realiza una solicitud HTTP/HTTPS al servidor web para acceder al recurso /index.html.
Paso 2: DNS: Antes de la conexión, el navegador consulta el DNS para obtener la dirección IP asociada al dominio www.ejemplo.com.
Paso 3: Conexión segura (HTTPS): Al ser una URL con HTTPS, el navegador inicia un proceso de handshake TLS para establecer una conexión segura.
Paso 4: Solicitud: El cliente envía una solicitud HTTP GET al servidor.
Paso 5: Servidor: El servidor procesa la solicitud, busca el archivo solicitado y lo responde con un código de estado (por ejemplo, 200 OK) y el contenido HTML.
Paso 6: Respuesta del cliente: El navegador recibe el contenido, lo interpreta y lo muestra al usuario.

Ejercicio 1.2: Comparación entre HTTP y HTTPS

Enunciado:

Completa la siguiente tabla comparando los protocolos HTTP y HTTPS. Luego responde: ¿por qué se recomienda el uso de HTTPS en sitios modernos?

Resolución:

Característica HTTP HTTPS
Puerto predeterminado 80 443
Cifrado de datos No Sí
Seguridad frente a ataques Baja Alta
Requiere certificado SSL/TLS No Sí
¿Por qué se recomienda HTTPS?
  • Protege la privacidad de los usuarios mediante cifrado
  • Genera confianza mostrando el candado en el navegador
  • Mejora el posicionamiento en Google (factor de ranking)
  • Es requisito para usar APIs modernas (geolocalización, notificaciones)
  • Evita advertencias de "sitio no seguro" en navegadores modernos

Ejercicio 1.3: Interpretación de una solicitud HTTP

Enunciado:

GET /productos.html HTTP/1.1 Host: www.mitienda.com User-Agent: Mozilla/5.0 Accept: text/html

Analiza la siguiente solicitud HTTP y responde las preguntas:

Preguntas:

  1. ¿Qué recurso se está solicitando?
  2. ¿Qué tipo de método HTTP se utiliza?
  3. ¿Qué protocolo se usa?
  4. ¿Cuál es el nombre del servidor?
  5. ¿Para qué sirve la cabecera User-Agent?

Resolución:

  1. Recurso solicitado: Se está solicitando el archivo productos.html.
  2. Método HTTP: El método es GET, usado para obtener datos.
  3. Protocolo: El protocolo especificado es HTTP/1.1.
  4. Nombre del servidor: El servidor es www.mitienda.com.
  5. Cabecera User-Agent: User-Agent informa al servidor sobre el navegador y sistema operativo del cliente, lo que puede ser útil para adaptar el contenido o registrar estadísticas.
