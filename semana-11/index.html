Modelo de Caja (Box Model)

El modelo de caja es un concepto fundamental en CSS que describe cómo los elementos HTML son estructurados y espaciados en una página web. Cada elemento es representado como una caja rectangular que contiene:

  • Contenido: El área donde se muestra el texto, imágenes u otros elementos
  • Relleno (Padding): El espacio entre el contenido y el borde
  • Borde (Border):strong> La línea que rodea el relleno y el contenido
  • Margen (Margin):strong> El espacio exterior que separa el elemento de otros elementos

Visualización del Box Model

Este es el contenido del elemento con padding, border y margin

Margen

El margen es un espacio transparente que rodea el borde de un elemento, separándolo de los demás elementos cercanos. Puede definirse con valores tanto positivos como negativos.

Propiedades de Margen:

  • margin-top - Margen superior
  • margin-right - Margen derecho
  • margin-bottom - Margen inferior
  • margin-left - Margen izquierdo
  • margin - Todos los márgenes simultáneamente
.container { border: 5px solid blue; margin: 40px; } .box { border: 5px solid rebeccapurple; background-color: lightgray; padding: 10px; height: 100px; margin-top: -40px; margin-right: 30px; margin-bottom: 40px; margin-left: 4em; }

Borde

El borde se encuentra ubicado entre el margen y el relleno de un elemento. Los bordes se pueden personalizar ampliamente, ya que cada uno de los cuatro lados puede tener su propio estilo, grosor y color.

Propiedades de Borde:

  • border-top, border-right, border-bottom, border-left
  • border-width, border-style, border-color
  • border - Todos los bordes simultáneamente
.container { margin: 40px; padding: 20px; border-top: 5px dotted green; border-right: 1px solid black; border-bottom: 20px double rgb(23 45 145); } .box { padding: 20px; background-color: lightgray; border: 1px solid #333333; border-top-style: dotted; border-right-width: 20px; border-bottom-color: hotpink; }

Relleno

El relleno es el espacio que se sitúa entre el borde de un elemento y su contenido. A diferencia del margen, el relleno no admite valores negativos. Además, cualquier fondo asignado al elemento se extiende también por el área de relleno.

Propiedades de Relleno:

  • padding-top - Relleno superior
  • padding-right - Relleno derecho
  • padding-bottom - Relleno inferior
  • padding-left - Relleno izquierdo
  • padding - Todos los rellenos simultáneamente
.box { border: 5px solid rebeccapurple; background-color: lightgray; padding-top: 0; padding-right: 30px; padding-bottom: 40px; padding-left: 4em; }

Contenido

Modelo de Caja Estándar

En el modelo de caja estándar, cuando se asignan valores a las propiedades width y height, estos determinan únicamente el tamaño del área de contenido de la caja, sin incluir el relleno, el borde ni el margen.

Modelo de Caja Alternativo

En el modelo de caja alternativo, el ancho especificado corresponde al tamaño total visible de la caja. El área de contenido se calcula restando del ancho total el espacio que ocupan el relleno y el borde.

Comparación de Modelos de Caja

Modelo estándar: width = contenido
Modelo alternativo: width = contenido + padding + border
.box { border: 5px solid rebeccapurple; background-color: lightgray; padding: 40px; margin: 40px; width: 300px; height: 150px; } .alternate { box-sizing: border-box; }

Propiedad Display

La propiedad display especifica cómo se debe representar el elemento. Entre sus valores más comunes:

Valor Descripción
block Elemento en bloque, comienza en nueva línea, ocupa todo el ancho
inline Elemento en línea, ignora width y height
inline-block Elemento en línea pero con propiedades de bloque
flex Contenedor flexible
grid Contenedor de cuadrícula
none Elemento no se muestra y no ocupa espacio

Propiedad Visibility

La propiedad visibility se utiliza para controlar la visibilidad de un elemento en la página.

Valor Descripción
visible Elemento visible normal
hidden Elemento oculto pero ocupa espacio
collapse Para tablas, oculta filas/columnas sin espacio

Propiedad Opacity

La propiedad opacity permite definir el nivel de transparencia de un elemento mediante un valor numérico entre 0 y 1.

Valor Descripción
0 Totalmente transparente
0.5 50% de transparencia
1 Totalmente opaco

Ejemplos de Opacity

Opacidad 0.5
Opacidad 0.25

Overflow

La propiedad overflow determina si el contenido que excede el tamaño asignado a un elemento debe ser recortado o si se deben mostrar barras de desplazamiento.

Valor Descripción
visible El contenido se muestra fuera de los límites
hidden El contenido que se desborda es recortado
scroll Se añaden barras de desplazamiento
auto Barras solo si el contenido lo requiere

Ejemplos de Overflow

Posicionamiento

La propiedad position define el método que se utilizará para colocar un elemento dentro del documento.

Valor Descripción
static Posición normal del flujo del documento
relative Relativo a su posición original
fixed Fijo en relación a la ventana del navegador
absolute Relativo al ancestro posicionado más cercano
sticky Comportamiento híbrido entre relative y fixed

Ejemplos de Posicionamiento

Elemento relative (desplazado 30px a la izquierda)
Elemento absolute (posicionado relativamente)
Elemento sticky (se pegará al hacer scroll)

Ejercicio 3.2.1: Box Model Card

Objetivo:

Comprender cómo funciona el modelo de caja y comparar diferentes configuraciones de padding, border y margin.

Código:

* { box-sizing: border-box; } body { font-family: system-ui, sans-serif; margin: 0; padding: 24px; } .wrap { display: flex; gap: 16px; flex-wrap: wrap; } .card { width: 280px; margin: 12px; padding: 16px; border: 4px solid #198754; border-radius: 8px; background: #f0fff4; } .ruler { height: 12px; background: repeating-linear-gradient(90deg,#ccc, #ccc 8px,#eee 8px,#eee 16px); }

Mini-retos:

  1. Cambia a content-box global y compensa anchos hasta que ambas tarjetas midan igual
  2. Añade outline para depurar sin alterar el "box model"
  3. Crea una tercera tarjeta sin margen entre tarjetas (usa gap del contenedor)

Ejercicio 3.2.2: Display vs Visibility vs Opacity

Objetivo:

Comparar display:none, visibility:hidden y opacity:0 en notificaciones, distinguir efectos en el flujo y la interacción del usuario.

Código:

.notice { padding: 12px; background: #fff3cd; border: 1px solid #ffec99; border-radius: 8px; margin-bottom: 12px; } .hidden { display: none; } .ghost { visibility: hidden; } .fade { opacity: 0; } function setState(mode) { const n = document.getElementById('notice'); n.className = 'notice'; if(mode === 'hidden') n.classList.add('hidden'); if(mode === 'ghost') n.classList.add('ghost'); if(mode === 'fade') n.classList.add('fade'); }

Comparación de Efectos:

Propiedad Ocupa espacio Es accesible al foco Lectores de pantalla
display:none No No No anunciado
visibility:hidden Sí No Generalmente omitido
opacity:0 Sí ¡Sí! Puede anunciarse (cuidado)

Mini-retos:

  1. Evita que opacity:0 reciba foco añadiendo pointer-events:none
  2. Convierte la notificación en componente accesible que se cierre con tecla Esc
  3. Agrega animación de entrada respetando @media (prefers-reduced-motion: reduce)
